# Engine (Historical Alias: Exec)

`runtime/engine/` is retained for compatibility of existing runtime artifacts.
Primary normative meaning is `exec`.

## Semantic role

This surface represents the execution plane:
- execution runtime behavior
- external-effect interaction model
- resource/provider/network/storage gate behavior
- execution-side deterministic controls

## Canonical artifact status

- `schema/engine_cortex.v1.json` is retained as historical artifact identity.
- Its semantic interpretation is `exec` runtime model, not a standalone engine ontology.

## Constraints

- `exec` is not sovereign authority.
- `exec` must remain subordinate to `core` governance.
- `exec` must not absorb `brain` cognition semantics.
